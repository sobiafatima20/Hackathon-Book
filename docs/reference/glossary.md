# Glossary of Robotics and AI Terms

This glossary provides definitions for key terms used throughout the Physical AI & Humanoid Robotics Technical Book.

## A

**Action Planning**: The process of determining a sequence of actions to achieve a specific goal, considering the current state and constraints.

**AI Agent**: An autonomous entity that perceives its environment and takes actions to achieve specific goals.

**ASR (Automatic Speech Recognition)**: Technology that converts spoken language into text, enabling voice command processing.

## B

**Behavior Tree**: A hierarchical structure used to organize and execute robot behaviors in a modular and maintainable way.

## C

**Cognitive Planning**: High-level planning that involves reasoning about complex tasks, goals, and strategies for achieving them.

## D

**Digital Twin**: A virtual representation of a physical object or system that mirrors its real-world counterpart.

**Docker**: A containerization platform that packages applications and their dependencies for consistent deployment.

## E

**Embodied AI**: Artificial intelligence systems that interact with the physical world through robotic bodies.

## F

**Forward Kinematics**: The computation of the position and orientation of a robot's end-effector based on joint angles.

## G

**Gazebo**: A physics-based simulation environment for robotics that provides realistic sensor simulation and environment modeling.

**GPGPU (General-Purpose computing on Graphics Processing Units)**: Using graphics processing units for computational tasks beyond graphics rendering.

## H

**Humanoid Robot**: A robot with a human-like body structure, typically including a head, torso, arms, and legs.

## I

**Inverse Kinematics**: The computation of joint angles required to achieve a desired position and orientation of a robot's end-effector.

**Isaac Sim**: NVIDIA's robotics simulation platform for developing and testing AI-powered robots.

**Isaac ROS**: GPU-accelerated ROS packages for robotics applications developed by NVIDIA.

## J

**Joint**: A connection between two rigid bodies that allows relative motion between them.

## L

**LiDAR**: Light Detection and Ranging - a remote sensing method that uses light in the form of a pulsed laser to measure distances.

**LLM (Large Language Model)**: A type of artificial intelligence model trained on large amounts of text data to understand and generate human language.

## M

**Middleware**: Software that provides common services and capabilities to applications beyond what's offered by the operating system.

## N

**Nav2**: The navigation stack for ROS 2, providing path planning and execution capabilities for autonomous mobile robots.

**Node**: In ROS, a process that performs computation. Nodes are the fundamental building blocks of ROS applications.

## P

**Perception Pipeline**: A sequence of processing steps that transforms raw sensor data into meaningful information for robot decision-making.

**Pseudocode**: An informal high-level description of the operating principle of a computer program or other algorithm.

## R

**Robot Operating System (ROS)**: A flexible framework for writing robot software that provides services designed for a heterogeneous computer cluster.

**ROS 2**: The second generation Robot Operating System providing communication infrastructure for robotic applications.

## S

**SDF (Simulation Description Format)**: An XML format for representing robot models and environments in Gazebo.

**SLAM (Simultaneous Localization and Mapping)**: The computational problem of constructing or updating a map of an unknown environment while simultaneously keeping track of an agent's location within it.

**Service**: In ROS, a communication pattern where one node sends a request and another node responds with a result.

## T

**Topic**: In ROS, a named bus over which nodes exchange messages.

## U

**URDF (Unified Robot Description Format)**: An XML format for representing robot models in ROS.

## V

**VLA (Vision-Language-Action)**: Systems that integrate visual perception, natural language understanding, and action execution for robotic applications.

**VSLAM (Visual SLAM)**: SLAM that uses visual sensors as the primary source of information for localization and mapping.

## W

**Whisper**: An automatic speech recognition (ASR) system developed by OpenAI.

## Y

**Yaw**: The rotation of an object around its vertical axis.

## Z

**ZMP (Zero Moment Point)**: A concept used in robotics to determine the stability of walking robots.
